Interface dragonfly.body.interfaces.IEgg

  • dragonfly.body.interfaces.IEgg

Description

IEgg defines core methods of Egg values operation.

Method Index

getValue(), setValue(), shoke()

Method Detail

shoke

public function shoke(start:Object, end:Object, duration:Number, delay:Number, equation:Function, equationArgs:Array, forceInit:Boolean, fps:Number, useFrames:Boolean):Void

This method usually don't need to be overwriten by childs classes in case you need to make a different use of some prop/value, try to call the super-method with the same arguments before, so the queue execution won't get broken.

Parameters

startStart value for the Egg transition.
endEnd value for the Egg transition.
durationDuration value for the Egg transition, in milleseconds.
delayDelay before start the Egg transition, in milleseconds.
equationEquation for the Egg transition.
equationArgsEquationArgs to be applied to the given Equation.
forceInitIf TRUE, the start value is initialized imediatelly, even if the tween has a delay.
fpsIf useFrames is false, you can set the refresh interval for the 'setInterval' mode (default value is 30).
useFramesIf true 'onEnterFreame' mode used, otherwise (false) 'setInterval' mode is used.

getValue

public function getValue():Object

These methods MUST to be overwriten by childs classes, it's obrigatory jusr because it can be such specific for each egg you create. That way, you can easilly create any kind of Egg you need... in other words: freedom. Check the existent eggs as examples to fill these two methods, take a breath, and follow your intuition.
